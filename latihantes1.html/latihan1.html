Dunia Programming

Programming adalah seni dan ilmu untuk membuat perangkat lunak. Dalam panduan ini,kita akan menjelajahi konsep,bahasa, dan praktik dalam dunia programming

1.Dasar-dasar Programming

  • Algoritma

    Algoritma adalah langkah-langkah logis untuk menyelesaikan masalah. Algoritma menjadi fondasi setiap program yang dibuat.

    Pseudocode
    Representasi algoritma menggunakan bahasa manusia atau kode semi-formal.
    Flowchart
    Diagram visual yang menunjukan alur logika program.
  • Struktur Data

    Struktur data adalah cara menyimpan dan mengorganisir data agar dapat digunakan secara efisien.

    • Array

      Koleksi elemen dengan tipe sama yang diakses melalui indeks.

    • Linked List

      Rangkaian elemen yang saling terhubung dengan pointer, berguna untuk penyisipan dan penghapusan data yang fleksibel.

    • Stack & Queue

      Stack mengikuti prinsip LIFO(Last In First Out), sedangkan Queue mengikuti FIFO(First In First Out).

2.Bahasa Pemrograman

  1. Bahasa Tingkat Tinggi

    Bahasa ini mudah dibaca manusia dan biasanya tidak bergantung pada perangkat keras tertentu.

    • Python

      Cocok untuk pemula,pengembangan web, dan science, dan scripting.

    • JavaScript

      Digunakan untuk pengembangan web interaktif, termasuk frontend dan backend(Node.js).

  2. Bahasa Tingkat Rendah

    Bahasa ini lebih dekat dengan mesin, seperti Assembly atau C, memungkinkan kontrol penuh terhadap hardware

    • C

      Bahasa klasik untuk sistem operasi dan embedded programming.

      Assembly

      Bahasa simbolik yang mewakili instruksi mesin secara langsung.

3.Paradigma Pemrograman

Object-Oriented Programming(OOP)
memodelkan program menggunakan objek dan kelas, memudahkan pemeliharaan dan pengembangan besar.
Functional Programming
Berfokus pada fungsi murni dan menghindari state yang berubah, populer di Haskell dan Scala.
Procedural Programming
Memecah program menjadi prosedur atau fungsi, banyak digunakan di C dan Pascal.

4.Praktik Baik dalam Programming

  • Code Readability

    • Menulis kode yang mudah dibaca dan dimengerti orang lain

  • Version Control

    menggunakan sistem seperti Git untuk melacak perubahan kode dan kolaborasi tim.

  • Testing

    Mengimplementasikan unit test, interogation test, dan functional test untuk memastikan kualitas kode.
